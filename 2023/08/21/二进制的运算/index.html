二进制的运算

简介

  二进制运算是计算机科学和电子工程等领域中基础的数学运算。它们通常涉及两个二进制数（0和1）的计算，并返回一个或多个二进制结果。最常见的二进制运算有加法、减法、乘法、除法，以及位运算（如与、或、非、异或 )
  在上一篇文章中，我们提到了计算机基本上只能理解0和1，这就引出了一个自然的问题：计算机是如何进行复杂的运算的呢？

计算机和运算

  首先，重要的是要明确一点：计算机本身并不会“做运算”，至少不是以人类传统认知的方式。实际上，计算机通过硬件逻辑门（如AND、OR、NOT等）来执行基础操作，这些操作在更高级别上组合成我们所熟悉的加法、减法、乘法等运算。

硬件逻辑门

  这些逻辑门是构建在晶体管之上的电路，它们接收一或多个二进制输入（0或1），并生成一个二进制输出。例如，AND逻辑门接收两个输入，只有当两个输入都为1时，输出才为1。

  在计算机科学和数学中，逻辑运算符如 AND（与）、OR（或）和 XOR（异或）在二进制运算中起着关键作用。下面是这些运算的文字描述：

AND（与）运算

  • 基本原则: 两个位都为1时，结果才为1。
  • 运算方式: 对两个二进制数进行按位比较，仅当两个相应位都为1时，结果位才为1。

OR（或）运算

  • 基本原则: 至少一个位为1时，结果就为1。
  • 运算方式: 对两个二进制数进行按位比较，只要有一个相应位为1，结果位就为1。

XOR（异或）运算

  • 基本原则: 两个位不相同时，结果为1。
  • 运算方式: 对两个二进制数进行按位比较，只有当两个相应位不相同时，结果位才为1。   这些逻辑运算符在计算机科学、电子工程以及其他信息处理领域中有广泛应用，包括但不限于条件判断、数据传输和加密等。理解这些基本运算是深入了解计算机工作原理的关键。

组合逻辑和算术运算

  通过这些基础的逻辑门，计算机硬件能够构建更复杂的“组合逻辑”结构，如加法器、乘法器等。例如，一个简单的二进制加法器会用到多个AND、OR和XOR逻辑门来实现两个二进制数的加法。

加法运算实现

例子一

  假设我们要实现十进制的8和6这两个数

  • 8的二进制表示是 1000
  • 6的二进制表示是 0110

  1. XOR操作（异或）：这步用于找出哪些位在加法运算中会发生变化。

    结果是1110。

  2. AND操作（与）：这步用于找出哪些位在加法中会产生进位。

    结果是0000，意味着没有进位。

  3. 左移进位：因为AND操作的结果是0000，说明没有进位。如果有进位，我们会将其左移一位。

    没有进位，所以这一步的结果还是0000。

  4. 最终结果：我们将XOR操作的结果与左移后的AND操作结果进行再次XOR操作。因为AND操作和左移结果是0000，再次XOR操作不会改变原来XOR操作的结果。

    最终结果为1110，即十进制的14.

例子二

  让我们用逻辑门（XOR 和 AND）来解释如何进行十进制的2和3这两个数的加法。

  • 2的二进制表示是 0010
  • 3的二进制表示是 0011

  1. XOR操作（异或）：这步用于找出哪些位在加法运算中会发生变化。

    XOR操作的结果是 0001。

  2. AND操作（与）：这步用于找出哪些位在加法中会产生进位。

    AND操作的结果是 0010，意味着我们有一个进位。

  3. 左移进位：我们将AND操作的结果左移一位。

    左移后，我们得到了 0100。

  4. 最终结果：我们将XOR操作的结果与左移后的AND操作结果进行再次XOR操作。

    最终结果是 0101，即十进制的5。

  通过这种方法，我们用XOR和AND逻辑门成功完成了2+3的二进制加法。这个过程模拟了计算机硬件如何进行加法运算。
通过这种方法，我们成功地用XOR和AND逻辑门完成了二进制的加法运算。这也是计算机硬件在底层执行加法的一种简化表达方式。

指令集

  在软件层面，计算机通过执行指令集中定义的基础运算来进行更高级的计算。这些指令在硬件层面被翻译成一系列逻辑门操作。
  常见的指令集（Instruction Set Architecture，ISA）可以分为以下几类：

CISC（Complex Instruction Set Computing）

  1. x86/x86_64: 这是Intel和AMD等公司用于桌面和服务器CPU的指令集，也是最常见的指令集之一。
  2. VAX: 由DEC公司开发，用于其早期的大型机。
  3. Motorola 68k: 用于一些嵌入式系统和早期的个人电脑（如Apple Macintosh）。

RISC（Reduced Instruction Set Computing）

  1. ARM: 用于大多数移动设备（如智能手机和平板电脑）以及嵌入式系统。
  2. MIPS: 主要用于嵌入式系统，如路由器。
  3. SPARC: 主要用于大型企业级服务器。
  4. PowerPC: 曾经用于Apple的Macintosh电脑，现在主要用于嵌入式系统和高端服务器。
  5. RISC-V: 开源的RISC指令集，用于各种应用，从嵌入式系统到高性能计算。

EPIC（Explicitly Parallel Instruction Computing）

  1. Itanium: 由Intel和HP共同开发，主要用于高端服务器。

其他

  1. CUDA: 由NVIDIA开发，专用于其图形处理器（GPU）。
  2. AVX: 用于现代Intel和AMD处理器的高级矢量扩展。
  3. NEON: ARM的SIMD（单指令多数据流）指令集。
  4. z/Architecture: 用于IBM的大型机。

  这些指令集是计算机硬件与软件之间的接口，不同的指令集有不同的特点和应用场景。例如，CISC指令集通常有更多的指令和更复杂的硬件实现，但它们可能在单个指令中完成更多的操作。与之相反，RISC指令集倾向于有更少、更简单的指令，但通常需要更多的指令来完成同样的任务。

总结

  所以，虽然计算机本身不会“做运算”，但它的硬件结构和软件编程允许它执行从最简单到最复杂的各种数学和逻辑操作。这一切都是建立在能够理解和处理二进制数（0和1）的基础上的。希望这篇文章能帮助你更深入地理解计算机是如何进行运算的。
